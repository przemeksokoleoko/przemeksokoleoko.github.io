What if a blog post didn't have to be a wall of text?

I've been thinking about a different format — using **mindmaps as the primary content unit**. Instead of paragraphs, the idea lives as a connected structure you can explore visually. This is the first experiment with that approach, powered by [markmap.js](https://markmap.js.org/).

# Mindmapping Blog

## The Idea
- Blog posts as interactive mindmaps
- Readers explore, not just read
- Ideas shown as relationships, not sequences

## Why Mindmaps
- Match how we actually think
- Non-linear structure
- Great for brainstorming & concept overviews
- Quick to scan, easy to revisit

## Why markmap.js
- Renders from plain Markdown
- Interactive — zoom & drag
- Works in static sites
- No backend needed

## This Post Is the Proof
- Written in Markdown
- Rendered as a mindmap
- Hosted on GitHub Pages
- Zero extra tooling

## What's Next
- More posts in this format
- Mix of mindmap + text posts
- Exploring topic-specific maps

The map above is fully interactive — zoom in, drag branches, explore freely.

This is format #1 of an ongoing experiment. If mindmaps can replace some blog posts, maybe they can replace a lot of things we usually write in linear prose.
